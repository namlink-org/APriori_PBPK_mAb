# A Priori PBPK Simulations of Monoclonal Antibodies — NAMLink

**A fully automated pipeline for predicting the pharmacokinetics of therapeutic antibodies in cynomolgus monkeys and humans, using only in vitro binding parameters.**

Part of the [NAMLink](https://github.com/NAMLink) initiative for open-source, animal-alternative drug development tools.

---

## What This Project Does

When a pharmaceutical company develops a new monoclonal antibody (mAb) drug, they need to predict how it will behave inside the body — how fast it distributes, how long it stays in the blood, and whether the dose is appropriate. This project automates those predictions using **Physiologically-Based Pharmacokinetic (PBPK) modelling**:

1. Takes in vitro binding measurements for 8 approved mAb drugs (measurable in a lab dish, no animals needed)
2. Automatically generates all simulation configuration files
3. Runs PBPK simulations in PK-Sim for both cynomolgus monkeys (NHP) and humans
4. Compares predicted blood concentration profiles against published clinical/preclinical data
5. Reports fold errors and generates PK profile charts for all 8 drugs × 2 species

The model captures **FcRn recycling** (why antibodies last weeks, not hours) and **Target-Mediated Drug Disposition** (TMDD — how target binding affects clearance at low doses), which are the two dominant mechanisms governing mAb pharmacokinetics.

Open `APriori_PBPK_mAb.Rproj` in RStudio first (ensures relative paths resolve correctly). Then render the three scripts **in order**:

```

```

Action: Click Render
Output: SimsOutputs/SimulationResults/<timestamp>/  (38 CSVs)
Time:   ~10–30 min depending on hardware (uses all available CPU cores)
```

**Step 3 — Process results and generate plots**
```

The pipeline runs **38 simulation scenarios** across two species:

| Species | Scenarios | Doses |
|---|---|---|
| Cynomolgus Monkey (NHP) | 14 | Driven by observed NHP study doses |
| Human | 24 | Driven by clinical trial doses from published literature |
| **Total** | **38** | |

Human scenarios span: Atezolizumab (5 doses), Avelumab (4), Bevacizumab (1), Golimumab (6), Nivolumab (6), Pembrolizumab (1), Tocilizumab (1).

> **Note:** Adalimumab is excluded from human simulations — available human data is subcutaneous (SC) 40 mg flat dose, while the PBPK model is IV-only. An IV–SC comparison would not be meaningful.

## Adding a New Drug

1. Confirm the target receptor has a `.pkml` model file in `00_Models/PKML_Files_Monkey/` and `PKML_Files_Human/`. If the drug targets a new receptor, Krina creates new pkml files in PK-Sim.
2. Add 6 rows to `01_Data/Monkey/Parameters.xlsx` and `01_Data/Human/Parameters_human.xlsx`.
3. Add observed NHP PK data to `01_Data/Monkey/mAb_data.csv`.
4. Add observed human PK data to `01_Data/Human/mAb_data_human.csv`.
5. Re-run Script 01 — the new drug is picked up automatically.

## Scientific Background

**PBPK (Physiologically-Based Pharmacokinetic) modelling** represents the body as a network of anatomical compartments (blood, liver, kidney, muscle, fat, etc.) connected by realistic blood flows. Unlike empirical PK models fitted to data, PBPK models can predict drug behaviour in untested species or patient populations using physiology and in vitro measurements alone.

**FcRn recycling:** Antibodies bind to the Neonatal Fc Receptor (FcRn) inside endosomes (cell recycling compartments). This rescues them from degradation, giving mAbs a plasma half-life of 2–3 weeks instead of hours. Kd(FcRn) controls this in the model.

**TMDD (Target-Mediated Drug Disposition):** At low doses, drug binding to its pharmacological target (e.g. PD-1 on T cells) is a significant elimination pathway. This causes nonlinear PK — the drug clears faster at low doses. The model includes a full TMDD module for each target.

**Fold Error (FE):** The primary acceptance criterion in PBPK validation. FE = Predicted ÷ Observed. Values between 0.5 and 2.0 indicate the model predicts within a factor of 2 of the observed value — the standard benchmark for a priori predictions.
